Clean HTML

Separation of content and display

HTML always had direct formatting features e.g. using the <font> element. But direct formatting was an obsolete idea even in the twentieth century. With the advent of CSS we never need to use the formatting elements so most of them were removed from HTML5. The separation of content and display produces more readable and maintainable code and also helps separate the roles in the development process.

Use of semantic elements

Earlier HTML versions already had semantic elements like <h1> , but HTML5 focuses on the semantic elements to get better maintainability and to allow creation of accessible websites.

A few semantic elements in HTML5

  • <h1>..<h6>
  • <p>
  • <header>
  • <footer>
  • <nav>
  • <section>
  • <article>
  • <aside>
  • <details>
  • <summary>
  • <ul>, <ol>, <dl>

A Simple responsive grid

1

This is a simple responsive grid implemented purely in CSS. Unlike commonly used grid frameworks we do not define a lot of width classes like col_1_3, col_1_4 etc.

2

The only class defined for this grid is cell. Add this class to your divs, sections or other elements to set up the grid.

3

The grid is displayed in a 2 column layout if the screen width is at least 34em. If the screen is narrower then the grid will fall back to 1-column display.

Please note that the flow of cells is different from what you may expect in a 2-column text. This grid was not designed for displaying text but for page layout.

4

There is also limited support for nesting cells. Nested layout breaks earlier to 1-column to avoid too narrow columns.
This behavior is also defined using media queries. This has some drawbacks but there is no better method.
Resize the browser window gradually and check how the layout changes!
